2. Автоматизация тестирования: Обзор и принципы
Цели автоматизации: повышение эффективности, снижение затрат времени, уменьшение человеческого фактора.
Когда применять автоматизацию: критические пути, повторяющиеся задачи, тестирование регрессий.
Преимущества и недостатки автоматизации тестирования.

3. Технологии автоматизации тестирования
Selenium (и его библиотеки: WebDriver, Selenide)
Cypress (быстрое и эффективное тестирование веб-приложений)
TestNG/JUnit (как структуры для организации тестов)
Appium (автоматизация тестирования мобильных приложений)
Katalon Studio (инструмент для низкокодовой автоматизации)
Cucumber (BDD для улучшения взаимодействия с бизнес-аналитиками)
Robot Framework (многофункциональный инструмент для различных типов тестирования)

4. Архитектуры и паттерны в автоматизации тестирования
Page Object Model (POM): преимущества использования при создании устойчивых тестов.
Screenplay pattern: современный подход к моделированию действий пользователя.
Data-Driven, Keyword-Driven, Behavior-Driven Development (BDD): различные подходы к организации тестов.

5. Интеграция автоматизации тестирования в CI/CD
Использование Jenkins, GitLab CI, TeamCity для автоматизации запуска тестов.
Взаимодействие с контейнерами через Docker и оркестраторами типа Kubernetes.
Мониторинг результатов тестов с использованием Allure и других репортинг инструментов.

6. Методы и стратегии тестирования производительности
Load Testing, Stress Testing, Spike Testing, Endurance Testing.
Инструменты для тестирования производительности: JMeter, Gatling, Locust.

7. Тестирование безопасности
Penetration Testing (ручное и автоматизированное)
Инструменты для тестирования безопасности: OWASP ZAP, Burp Suite, Nmap.
Актуальные вопросы: SQL-инъекции, XSS-атаки, авторизация и аутентификация.
